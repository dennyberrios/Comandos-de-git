Comandos do git.

remote

git remote

Lista todos os repósitorios remoto que o repositório local conhece.

git remote add origin (link do github)

O comando git remote permite criar, ver e excluir conexões com outros repositórios. As conexões remotas são mais parecidas com marcadores em vez de links diretos para outros repositórios.

git remote -v

Mostra todos indereços.

git remote rename nomo antigo novo nome

Renomea o repositório. Exemplo: git remote rename nome antigo(origin) nome novo (Local).

mesclagem

git marge nome da outra branch

Cria um commit de junção de duas branch. para salvar a mensagem e confirma digite :x pressione enter.

git rebase nome da branch

Traz os commits de uma branch para a outra.

alterações no arquivo e commits

git checkout -- nome do arquivo

Descarta alterações.

git reset head nome do arquivo

desfaz as alterações que já foi marcadas para commita.

git revert Hash do commit

Desfaz o commit.

Obs: Em sua essência, o sistema de controle de versão do Git é um sistema de arquivos endereçável por conteúdo. Ele usa a função de hash para nomear o conteúdo. para encontra o Hash basta digita no seu terminal dentro da pasta do arquivo, git log --oneline. para mais informações sobre o hash acesse aqui.

salva e pega os arquivos

git stash

Salva todas as alterações para um local temporario.

git stash list

Mostra uma lista de tudo que esta salvo no stash.

git stash apply 0

Aplica as modificações.

git stash drop

Remove a lista.

git stash pop

Pega e remove a ultima alteração.

git checkout Hash

Volta ate o commit de hash.

Obs: Em sua essência, o sistema de controle de versão do Git é um sistema de arquivos endereçável por conteúdo. Ele usa a função de hash para nomear o conteúdo. para encontra o Hash basta digita no seu terminal dentro da pasta do arquivo, git log --oneline. para mais informações sobre o hash acesse aqui.

compara commits

git diff

O git diff é um comando Git multiuso que, quando executado, realiza uma função de comparação nas fontes de dados Git. Essas fontes de dados podem ser commits, ramificações, arquivos e outros.

git diff Hash..Hash

Compara dois commits.

Obs: Em sua essência, o sistema de controle de versão do Git é um sistema de arquivos endereçável por conteúdo. Ele usa a função de hash para nomear o conteúdo. para encontra o Hash basta digita no seu terminal dentro da pasta do arquivo, git log --oneline. para mais informações sobre o hash acesse aqui.

cria e sobi tag

git tag -a " nome da versão "

Cria um marco na aplicação. exemplo: git tag -a V0.1.0

git tag -a " nome da versão " -m " mensagem "

O -m adiciona uma mensagem na tag antes do commit. exemplo: git tag -a V0.1.0 -m " Primeira versão do projeto "

git push nome da branch versão

Envia a tag. exemplo: git push master V0.1.0

Comandos basicos do terminal

dir

O comando dir exibe, em formato largo, uma lista alfabética dos nomes de arquivo correspondentes em cada diretório e pausa cada vez que a tela é preenchida até que você pressione qualquer tecla para continuar.

cd nome da pasta

Entra na pasta.

cd ..

volta uma pasta.

ls

Mostra o que tem dentro da pasta.

mkdir nome da pasta

Nomea e cria uma pasta.

Configuração do usuario no git.

git config --global user.name "seu nome"

Configurar o nome de usuário globalmente no Git para todos os repositórios.

git config --global user.email "seu email cadastrado no github"

Configurar o e-mail de usuário globalmente no Git para todos os repositórios.

git config --local user.name "seu nome"

Configurar o nome de usuário local no Git para um repositório.

git config --local user.email "seu email cadastrado no github"

Configurar o e-mail de usuário local no Git para um repositório.

git config --list

Lista todas as configurações que o Git conseguir encontrar naquele momento.

branchs

git branch

Ver todas as branch

git branch nome da branch

Cria uma branch com o nome escolhido

git checkout nome da branch

Muda para outra branch

git checkout -b nome da branck

Cria e muda para a branck criada

git branch -m main

O comando git branch permite criar, listar, renomear e excluir ramificações. Ele não permite alternar entre as ramificações ou reunir um histórico bifurcado de novo.

Subir projetos no github.

git init

O comando git init cria um novo repositório do Git. Ele pode ser usado para converter um projeto existente e não versionado em um repositório do Git ou inicializar um novo repositório vazio.

git init --bare

A marcação --bare cria um repositório que não tem um diretório ativo, tornando impossível editar arquivos e fazer commit de alterações nesse repositório. Você cria um repositório vazio para git push e git pull, mas nunca faz o commit direto para ele.

git add .

O comando git add adiciona uma alteração no diretório ativo à área de staging. Ele diz ao Git que você quer incluir atualizações a um arquivo específico no próximo commit. No entanto, git add não tem efeito real e significativo no repositório — as alterações não são gravadas mesmo até você executar git commit .

git status

Mostra o statu da aplicação no repositório local.

git commit -m " mensagem "

O comando git commit captura um instantâneo das mudanças preparadas do projeto no momento. Os instantâneos com commit podem ser considerados versões "seguras" de um projeto, o Git nunca os altera, a menos que você peça a ele.

git push -u origin main

O comando git push é usado para enviar o conteúdo do repositório local para um repositório remoto.

Comando de logs

git log

O comando git log é a ferramenta básica do Git para explorar o histórico de um repositório. É usado quando se precisa encontrar uma versão específica de um projeto ou saber quais alterações vão ser implantadas por meio do merge de uma ramificação de recurso.

git log -p

Mostra as informações do commit e as alterações.

git log -n 2

Mostra os dois ultimos commits.

git log --oneline

A opção --oneline mostra informações sobre os commits em uma única linha, facilitando a navegação rápida entre commits.

git log --graph

Mostra as linhas específicas de desenvolvimento.

Clona conteúdo do github.

git clone ( link do repositorio )

O git clone é usado sobretudo para apontar para um repositório existente e fazer um clone ou cópia deste repositório no novo diretório, em outro local. O repositório original pode estar localizado no sistema de arquivos local ou em protocolos com suporte a acesso por máquinas remotas.
existentes em um repositorio remoto.

Baixa apenas as atualizações.

git pull

O git pull é um comando do Git usado para atualizar a versão local de um repositório a partir de uma versão remota. Ele é um dos quatro comandos que solicita a interação com a rede pelo Git.
